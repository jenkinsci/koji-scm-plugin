If checked, the freshly checkout build/NVR is checked again against processed.txt after download.

  • By default, it is NOT selected
  • If there is nothing to checkout - eg the NVR was added to processed.txt in meantime, the build gracefully stops
  • If the build was started by build-now, and the NVR will be found in processed.txt (which it most likely will), the build fails
  • This is safety belt for cases, when koji plugin get mad, and keep building same NVR in turns
    • That happens if job runs longer then scheduling pulls, and "KojiSCM skip pooling if job runs" is off.
    • That happens build now is pressed to often
    • That happens when checkout takes longer then the scheduling pulls
    • the build-now button will stop work as it is used to
  • This is temporary switch before better solution is found
